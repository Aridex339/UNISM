# Mitwirken am Projekt

Schön, dass du dabei bist! Um die Qualität des MVPs hochzuhalten, beachte bitte folgende Richtlinien:

## 🛠 Workflow
1. Erstelle ein **Issue**, bevor du größere Änderungen vornimmst.
2. Nutze für neue Features den Branch-Namen: `feat/dein-feature`.
3. Erstelle einen **Pull Request** gegen den `main` Branch.

## 📝 Commit Messages
Wir nutzen klare Nachrichten, um die Historie lesbar zu halten:
- `feat:` für neue Funktionen
- `fix:` für Fehlerbehebungen
- `docs:` für Änderungen an der Dokumentation

## 💬 Fragen?
Nutze das Issue-Template "Frage / Support" oder diskutiere direkt in den Pull Requests.
